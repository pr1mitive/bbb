## 🔄 業務フロー概要

### 発注書作成フロー (30分)

```
1. カスタムビュー「発注登録画面」を開く
2. 基本情報入力
   - 発注元・発注先を選択
   - 発注日・件名を入力
   - 通貨・税率を選択
3. 明細入力
   - 方法A: 見積から取込
   - 方法B: 手動入力
4. 案件配分設定 (必要な場合)
   - 各明細の「+ 案件配分」をクリック
   - 案件番号と数量を入力
   - または Excel 形式で貼り付け
5. 金額確認
6. 登録 (下書き or 本登録)
```

### ERP連携フロー (20分)

```
1. 発注書登録 (ステータス: 発注済み)
2. ERPテーブル自動生成
   - 案件配分を展開
   - 各案件ごとに行を生成
3. CSV出力 (今後実装)
4. ERPシステムに取込
5. 連携完了フラグ更新
```

## 💡 重要なポイント

### ユーザー向け

1. **案件配分は登録時のみ**
   - 後から追加・変更できません
   - 最初に必ず設定してください

2. **配分数量は必ず一致させる**
   - 配分合計 = 明細数量
   - 不一致の場合は警告が表示されます

3. **見積参照を活用**
   - 見積データがあれば必ず使用
   - 入力ミスを大幅に削減できます

4. **Excel貼り付けが便利**
   - タブ区切り (TSV形式) で貼り付け
   - 複数案件を一度に入力できます

### システム管理者向け

1. **JavaScript アップロード順序厳守**
   ```
   1. config.js
   2. utils.js
   3. masterData.js
   4. calculator.js
   5. styles_updated.js
   6. customView_part1.js
   7. customView_part2_updated.js
   8. customView_part3_updated.js
   9. projectAllocation.js
   10. erpBuilder.js
   ```

2. **App ID の正確な設定**
   - config.js で環境に合わせて変更
   - カスタムビューIDも忘れずに設定

3. **定期メンテナンス**
   - 月次でマスタデータを整備
   - 四半期でデータ整合性チェック
