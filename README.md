## 🎯 ユースケース

### ケース1: 通常の発注 (案件配分なし)

**シナリオ**: 消耗品を10個発注

**手順**:
1. 発注登録画面を開く
2. 基本情報を入力
3. 明細行を追加 (アイテムA、数量10)
4. 登録

**結果**:
- `po_items`: 1行 (数量10)
- `erp_items`: 1行 (案件番号空白、数量10)

### ケース2: 案件配分あり発注

**シナリオ**: 部品を10個発注、案件P001に6個、P002に4個

**手順**:
1. 発注登録画面を開く
2. 基本情報を入力
3. 明細行を追加 (アイテムA、数量10)
4. 「+ 案件配分」ボタンをクリック
5. P001: 6、P002: 4 を入力
6. 登録

**結果**:
- `po_items`: 1行 (数量10、案件表示「P001(6), P002(4)」)
- `erp_items`: 2行
  - 行1: 案件P001、数量6
  - 行2: 案件P002、数量4

### ケース3: 見積からの一括取込

**シナリオ**: 見積データを使って5明細の発注書を作成

**手順**:
1. 発注登録画面を開く
2. 発注先を選択
3. 「📋 見積参照」ボタンをクリック
4. 見積を検索・選択
5. 「この見積を取込」ボタンをクリック
6. 5明細が自動入力される
7. 必要に応じて案件配分を設定
8. 登録

**結果**:
- 5行の明細が一度に入力される
- 通貨も見積に合わせて自動設定

## 🌟 ベストプラクティス

### 発注書作成

1. **マスタデータを最新に保つ**
   - 月初にマスタを見直し
   - 新規取引先は事前登録

2. **案件配分は慎重に**
   - 配分合計と明細数量を必ず確認
   - Excelで事前に計算してから貼り付け

3. **見積参照を活用**
   - 見積データがあれば必ず使用
   - 入力ミスを減らせる

### データ管理

1. **定期的なバックアップ**
   - 月次でレコードをCSV出力
   - 設定変更前にアプリ設定を書き出し

2. **下書きは早めに完了**
   - 1週間以内に完了または削除
   - 放置しない

3. **ERP連携は即日**
   - 発注登録と同日にCSV出力・取込
   - タイムラグを最小化
